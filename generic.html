Project 1: Portal and modules for a Learning Management System

This project is comprised of content strategy, architecture, and research for knowledge base reorganization and completed user guide web pages for a LMS and module packages. The details of this project are confidential and not available for viewing.

Project objective

To complete the WCAG 2.1 compliant redesign of a private Learning Management System and module packages. To make the LMS and module packages accessible, all text, images, UI interactions, page layouts, and page navigation had to be redesigned.

My contribution

  • Working with an accessibility specialist to redesign the navigation logic of all pages within the modules and LMS, and referencing the WCAG. 2.1 document when I designed the navigation logic of the new knowledge base that contained all customer-help documentation.
  • Writing UI documentation of both the legacy and newly designed LMS and module packages that included accessibility functionality for internal use by engineers. I developed a content calendar so I could write the legacy documentation for the engineers and the new documentation of the redesigned products for archival purposes. I maintained all documentation in a Confluence space that I structured for company-wide use.
  • Auditing the user manuals produced for the legacy products and producing benchmarking analyses and comparative reports to determine technical writing best practices for designing a new knowledge base and customer success content.
